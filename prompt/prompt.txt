Implement a **Visual Drag-and-Drop Workflow Orchestration Designer** that allows users to orchestrate existing Scout workflows, AI actions, human approvals, and external triggers into a complete business process.

## Objective

Transform Scout from a workflow player into an enterprise orchestration platform where users can visually build business processes by connecting reusable nodes.

The orchestration engine should execute nodes sequentially or conditionally while sharing a common execution context.

---

## Core Concepts

### Orchestration

An orchestration is a graph consisting of nodes and connections.

Each orchestration has:

* Name
* Description
* Version
* Status (Draft / Published)
* Variables
* Execution Context
* Trigger
* Nodes
* Connections

---

## Designer

Create a drag-and-drop canvas.

Features:

* Zoom
* Pan
* Grid
* Auto alignment
* Multi-select
* Copy/Paste
* Undo/Redo
* Delete
* Validation
* Save Draft
* Publish

Each node should have configurable properties.

Connections should allow branching.

## Execution Context

Every node should share one execution context.

Example:

{
customer,
invoice,
email,
extractedData,
aiResult,
workflowOutputs
}

Nodes can read and update this context.

Avoid passing dozens of parameters between nodes.

---

## Variable Mapping

Support expressions such as:

{{customer.name}}

{{invoice.number}}

{{email.subject}}

{{workflow.CreateInvoice.invoiceId}}

## Extensibility

Design the node system using a plugin architecture.

Each node should implement a common interface such as:

* validate()
* execute()
* serialize()
* deserialize()
* renderProperties()

Adding a new node type should require minimal changes to the core engine.

## Implementation Requirements

* Reuse the existing Scout workflow engine instead of duplicating execution logic.
* Workflow nodes should execute existing workflows exactly as they do today.
* Reuse the existing active AI provider for AI Extraction and AI Decision nodes.
* Reuse the existing analytics, self-healing, versioning, and approval infrastructure wherever possible.
* Keep the orchestration engine modular, extensible, and independent of specific AI providers or external services.

The final architecture should position Scout as an enterprise AI orchestration platform capable of combining workflows, AI reasoning, human approvals, and business rules into complete end-to-end business processes.
But, remember, existing modules must not be broken. This should be a different page.
