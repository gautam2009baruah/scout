API TRIGGER PROMPT

Implement API Trigger for Orchestration Studio.

Behavior:

* API trigger allows authenticated systems to start an orchestration using REST API.
* Add endpoint:

POST /api/orchestrations/{orchestrationId}/execute

* Authentication should use existing auth system/API key system.
* Trigger configuration should include:

  * allowed API clients
  * required request schema
  * optional rate limit
  * enabled/disabled
* Validate request payload before execution.
* Store payload in execution context:

{
"trigger": {
"type": "api",
"clientId": "...",
"requestBody": {},
"requestedAt": "timestamp"
}
}

Requirements:

* Support API key or OAuth client if existing system supports it.
* Add API documentation panel in UI.
* Add request/response examples.
* Return execution ID and status.
* Log all API trigger attempts.

WEBHOOK TRIGGER PROMPT

Implement Webhook Trigger for Orchestration Studio.

Behavior:

* Webhook trigger exposes a unique endpoint for an orchestration.
* Trigger configuration should include:

  * generated webhook URL
  * secret token
  * allowed HTTP methods, default POST
  * optional IP allowlist
  * payload schema validation
  * enabled/disabled
* Incoming webhook should validate secret before execution.
* Support secret through header:

X-Scout-Webhook-Secret: <secret>

* Store incoming request body in context:

{
"trigger": {
"type": "webhook",
"headers": {},
"query": {},
"body": {},
"receivedAt": "timestamp"
}
}

Requirements:

* Never expose secret after generation.
* Allow secret rotation.
* Reject invalid signatures/secrets.
* Return execution ID after successful trigger.
* Log accepted and rejected webhook calls.
* Add sample curl command in UI.
